컨테이너 CPU/메모리 사용량 확인 명령어 모음 (Docker ~ Kubernetes)

컨테이너 CPU/메모리 사용량 확인 명령어 모음 (Docker ~ Kubernetes) 컨테이너의 리소스 이슈(메모리 누수, OOMKill, CPU 스로틀링, 노드 압박 등)를 빠르게 진단하려면 “어디에서(호스트/컨테이너/k8s)” “무엇을(실사용/제한/요청)” 보고 있는지 구분하는 게 중요하다. 이 글은 단일 Docker 환경 → Kubernetes 환경으로 이어지며 CPU/메모리 사용량을 확인하는 명령어를 정리한다. 0) 용어/관점 짧게 정리 Usage(사용량): 현재 실제로 쓰는 CPU/메모리 Request(요청): 스케줄링 기준(이 정도는 필요해요) Limit(제한): 넘으면 제한/종료(OOMKill 등)될 수 있는 상한 Working Set Memory: 캐시 등을 제외한, “실제로 압박을 주는 메모리”에 가까운 값(도구마다 정의 차이 있음) CPU throttling: CPU limit에 의해 실행이 지연되는 현상 (사용량이 낮아 보여도 성능 저하 가능) 1) Docker 단독 환경 1-1) 전체 컨테이너 사용량 한눈에 보기: docker stats docker stats 실행 중인 컨테이너들의 CPU%, MEM 사용량/제한, NET I/O, BLOCK I/O를 실시간으로 보여준다. 특정 컨테이너만 보고 싶으면 이름/ID를 붙인다. docker stats <container_name_or_id> 한 번만 출력하고 종료 docker stats --no-stream 1-2) 컨테이너 메모리 제한/설정 확인: docker inspect docker inspect <container> --format '{{json .HostConfig.Memory}}' docker inspect <container> --format '{{json .HostConfig.NanoCpus}}' Memory는 바이트 단위 제한값(0이면 제한 없음). NanoCpus는 CPU 제한(예: 2000000000 = 2 CPU). 제한이 없다면 컨테이너의 메모리 사용량은 호스트 메모리/커널 정책에 영향을 크게 받는다. ...
